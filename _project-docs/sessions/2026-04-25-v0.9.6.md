### v0.9.6 — UVC Camera Fixes + CameraStatusPanel + Crash Fix

**UVC event name fix:**
- `UVCModule.kt` — corrected all event name strings to match JS listener names:
  - `"UVCFrame"` → `"onFrame"`
  - `"UVCDisconnected"` → `"onCameraDisconnected"`
  - Added `sendEvent("onCameraConnected", null)` after connect resolves
  - Added `sendEvent("onCameraFormats", supported)` — emits supported sizes JSON after `camera.open()`
  - Added `getSupportedFormats()` `@ReactMethod`
- `lib/thermal/uvcCamera.ts` — added `onCameraFormats` listener + `getSupportedFormats()` async function

**Hardware reference document:**
- Created `_project-docs/hardware-references.md` — all 6 hardware links, hardware stack diagram, UVC format descriptors (YUYV/Y16/GREY/RGB565/BGR3), Y16 byte format + Kelvin conversion, libuvccamera constants, known Y16 integration gap, PureThermal + Lepton specs, USB device filter confirmation

**CameraStatusPanel:**
- Added to `app/(clinic)/live-feed.tsx` — replaces static status bar
- Features: animated pulsing dot, live FPS counter, Y16 sanity check warning banner (<50% pixels in 10–60°C range), retry button (`retryKey` re-triggers camera setup useEffect), format debug row from `onCameraFormats` event

**Crash fix — live-feed screen:**
- **Root cause 1:** `UVCModule.kt` tried `setPreviewSize(160, 120, 6)` — mode 6 is not a valid AAR Java constant; unhandled JNI exception killed the process; crash was 100% reproducible because camera was always connected on navigate
- **Fix:** `listOf(6, 0, UVCCamera.DEFAULT_PREVIEW_MODE)` → `listOf(UVCCamera.FRAME_FORMAT_YUYV, UVCCamera.DEFAULT_PREVIEW_MODE)`
- **Root cause 2:** `Animated.loop` in `CameraStatusPanel` had no cleanup → orphaned animation loop in Hermes after unmount → secondary crash
- **Fix:** `let loop = null; loop = Animated.loop(...); loop.start(); return () => { loop?.stop(); pulseAnim.stopAnimation(); };`
- Release APK rebuilt successfully
