# Session — v0.9.1
**Date:** 2026-04-07

## What We Did

### Thermal Image + CSV File Import
Added the ability to import a thermal image (JPG/PNG) and a temperature CSV file directly from the device, bypassing the need for a connected camera. Implemented across all three thermal-capable screens:

- **Clinic live-feed** (`app/(clinic)/live-feed.tsx`): "or import from files" divider section below the capture controls. Two cards side by side — Thermal Image (opens file picker, shows thumbnail preview) and Temperature CSV (parses to matrix, drives ThermalMap in real time). "Use Imported Data" CTA locks the frame and routes to clinical-data exactly like a live capture.
- **Offline live-feed** (`app/(offline)/live-feed.tsx`): Identical two-card section. CSV matrix encoded via `matrixToStorageB64()` for SQLite storage; routes to offline save screen.
- **Patient dashboard** (`app/(patient)/index.tsx`): Three picker cards inside the "Latest Thermal Scans" card — Thermal Image, Left CSV, Right CSV. Imported matrices replace mock thermal thumbnails in real time. Reference image shown as a full-width preview strip below the thumbnails.

### New utilities in preprocessing.ts
- `parseCsvMatrix(csv)` — parses comma/tab-separated rows of °C floats into `ThermalMatrix`; skips non-numeric header rows; supports any dimensions
- `matrixToStorageB64(matrix)` — encodes `number[][]` as base64 JSON for local SQLite (non-Y16 capture path)
- `parseStoredMatrix(b64)` — auto-detects JSON-encoded matrix vs raw Y16 B64 frame; used in sync upload to handle both types

### sync.tsx updated
- `parseY16Frame` replaced with `parseStoredMatrix` so that CSV-imported captures sync correctly to Supabase (previously would have thrown on JSON-encoded matrices)

### Bug fix
- `app/(patient)/index.tsx` — `colors.background` → `colors.bg` (invalid token, caused TypeScript error)
