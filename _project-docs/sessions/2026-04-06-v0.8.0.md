- Replaced all mock BLE code in `pairing.tsx` with real implementations

- Extended `useDeviceStore` with `cameraSource`, `wifiIp`, `wifiPort`
- Added `CameraSource = "uvc" | "wifi"` type
- Fully rewrote `app/(clinic)/pairing.tsx`: Active Source card, FLIR info section, ESP32 WiFi section (IP/port/test/connect), BLE Discovery (real scan + device list)
- Updated `app/(clinic)/live-feed.tsx`: camera setup `useEffect` now branches on `cameraSource`; WiFi path connects WebSocket; UVC path uses existing bridge; FPS badge labels camera source
- Added BLE Android permissions to `AndroidManifest.xml`
- Added all new UI strings to `constants/strings.ts` S.pairing

### Manual Actions Required
- **`npx expo run:android`** — must be run to rebuild the native app with `react-native-ble-plx`; Metro/Expo Go will not work until this is done
- **ESP32 firmware** — must implement:
  - BLE advertising name: `ESP32-Thermal` (prefix match)
  - BLE service UUID: `0000ffe0-0000-1000-8000-00805f9b34fb`
  - BLE characteristic UUID: `0000ffe1-0000-1000-8000-00805f9b34fb` (readable, returns WiFi IP as UTF-8)
  - WebSocket server on port 8080 at `/stream` path
  - Binary frame format: `0x54 0x4D` + uint16 width LE + uint16 height LE + 2 reserved bytes + (W×H) uint16 LE pixels (value = temp × 100)
  - Optional: `/ping` endpoint for connection test
- **UVCModule.kt** — still a stub; real `libuvccamera-release.aar` needs to be added to `android/app/libs/`
