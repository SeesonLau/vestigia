## What We Did

### Patient Settings Screen
- `app/(patient)/settings.tsx` was empty (1 line) — causing "missing default export" crash
- Built proper settings screen: Edit Profile, Change Password, Notifications, Sign Out rows

### SafeAreaView Warning
- `ScreenWrapper.tsx` was using deprecated `SafeAreaView` from `react-native`
- Replaced with `SafeAreaView` from `react-native-safe-area-context`
- No manual `SafeAreaProvider` needed — Expo Router provides it at root

### Error Handling Upgrade
- Added graceful handling for 504/503/502, network timeouts, and fetch failures
- All raw Supabase/server error text now replaced with user-friendly messages

### Email Confirmation — Deep Link Flow
**Problem:** Clicking the confirmation email link on a PC shows a browser error because `vestigia://` is not a web address.

**Solution — Supabase Edge Function redirect page:**
- Created `supabase/functions/auth-redirect/index.ts`
- Serves a Vestigia-themed HTML page
- On **mobile**: auto-fires `vestigia://confirm` to open the app; fallback "Open App" button after 2.5s
- On **desktop**: shows "Open on Your Phone" with MOBILE APP ONLY badge
- `emailRedirectTo` in `signUp` now points to this Edge Function URL

### Account Activated Screen
- Created `app/(auth)/account-activated.tsx`
- After confirmation deep link fires on mobile, app routes here instead of directly to dashboard
- Shows teal checkmark, "Account Activated" heading, "Sign In" button → routes to login
- Deep link handler simplified — no longer needs `routeByRole()`, just navigates to this screen
- Handles both implicit flow (`#access_token=...&type=signup`) and PKCE flow (`?code=...`)
